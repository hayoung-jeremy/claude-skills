# Work Tracker: 데일리 작업 기록 자동 생성

## 역할

개인 노션 Daily DB(SSoT)의 기록을 분석하여, 회사 통합 태스크의 인라인 '데일리 작업 기록' 데이터베이스에 구조화된 작업 기록을 자동 생성해요.

## 설계 원칙

**Single Source of Truth = 개인 노션 Daily DB**
- 모든 데이터는 개인 Daily DB → 회사 시스템 방향으로만 흐름
- 회사 구조가 변경되어도 개인 기록은 영향받지 않음
- 회사 DB는 "출력 대상"이지 "입력 소스"가 아님

## 핵심 ID 참조

| 항목 | ID |
|------|-----|
| Daily DB | `collection://31119aac-27e4-8008-bb4b-000bd838f764` |
| Notes DB | `collection://31119aac-27e4-80d9-a83c-000be57af066` |
| 전체 프로젝트 DB | `collection://0ffbed16-cd0a-4381-b293-b3e630f1d06d` |
| 통합 태스크 DB | `collection://db1ed8bf-4edf-4a87-b5bd-8774ce361a73` |
| 김하영 Notion User ID | `310d872b-594c-812e-b436-0002c6a46fa9` |

## 1단계: 대상 날짜 및 Daily 페이지 조회

아래 우선순위로 대상을 결정해요:
1. 사용자가 날짜를 명시한 경우 → 해당 날짜
2. `$ARGUMENTS`에 날짜가 있는 경우 → 해당 날짜
3. 아무 지정 없는 경우 → 오늘 날짜

### Daily 페이지 조회

```sql
SELECT url, "Name", "date:Date:start"
FROM "collection://31119aac-27e4-8008-bb4b-000bd838f764"
WHERE "date:Date:start" = '{대상_날짜}'
```

페이지가 없으면 사용자에게 알리고 종료해요.

`notion-fetch`로 Daily 페이지 내용을 읽어요. `## todo` 섹션에서 태스크별 작업 항목을 추출해요.

### Daily 페이지 파싱

Daily 페이지의 `## todo` 섹션 구조:
```markdown
## todo
* [태스크명A]
	- [x] 완료한 작업1
	- [x] 완료한 작업2
	- [ ] 미완료 작업
* [태스크명B]
	- [x] 완료한 작업
### misc.
- 기타 업무
```

각 `* [태스크명]` 헤더 아래의 체크박스 항목들을 해당 태스크에 매핑해요.
- `[x]` 항목 → 완료된 작업
- `[ ]` 항목 → 진행중/미완료 작업

`### misc.` 아래 항목은 태스크에 매핑하지 않아요 (작업 트래커 대상 아님).

## 2단계: Notes DB 보조 정보 수집

해당 날짜 전후로 생성된 Notes가 있는지 조회해요:

```sql
SELECT url, "Name", "생성일시:Date:start"
FROM "collection://31119aac-27e4-80d9-a83c-000be57af066"
WHERE "생성일시:Date:start" = '{대상_날짜}'
```

Notes가 있으면 `notion-fetch`로 내용을 읽고, `분류` 태그를 기반으로 어떤 태스크와 관련되는지 매칭해요.
Notes 내용 중 `## 요약`이나 `## 작업 내용` 섹션의 핵심 내용을 작업 트래커에 보충 정보로 활용해요.

## 3단계: 통합 태스크 매칭

### 내 진행중 태스크 조회

```sql
SELECT url, "태스크명", "상태", "담당자", "프로젝트", "단계"
FROM "collection://db1ed8bf-4edf-4a87-b5bd-8774ce361a73"
WHERE "상태" NOT IN ('완료', '아카이빙')
```

결과에서 `담당자` 필드에 `310d872b-594c-812e-b436-0002c6a46fa9`가 포함된 행만 필터링해요.

### 매칭 규칙

Daily의 `* [태스크명]` 태그와 통합 태스크 DB의 `태스크명`을 비교해요:
- 부분 문자열 매칭: 태그명이 태스크명에 포함되거나, 태스크명이 태그명에 포함
- 예: Daily `[Create/Variation]` ↔ DB `Create/Variation 모델 학습 및 검증`

매칭되지 않는 Daily 항목은 스킵하고 사용자에게 알려요.

## 4단계: 작업 트래커 DB 탐색

각 매칭된 통합 태스크 페이지에서 인라인 '데일리 작업 기록' DB를 찾아요.

### 탐색 방법

1. `notion-fetch`로 태스크 페이지를 조회해요
2. 페이지 내용에서 인라인 데이터베이스를 찾아요 (보통 `<data_source ...>` 태그로 표현됨)
3. 데이터베이스 이름이 "데일리 작업 기록"인 것을 선택해요
4. 해당 DB의 `collection://` ID를 추출해요

⚠️ **각 태스크마다 인라인 DB의 collection ID가 다릅니다.** 하드코딩 불가 — 반드시 동적으로 탐색해야 해요.

### DB를 찾지 못한 경우

- 태스크 페이지에 인라인 DB가 없으면 → 사용자에게 알리고 해당 태스크 스킵
- DB 이름이 다를 수 있음 → "작업 기록", "작업 트래커", "데일리" 등 유사 이름도 검색

## 5단계: 작업 트래커 항목 생성

### 중복 확인

먼저 해당 날짜의 기록이 이미 있는지 확인해요:

```sql
SELECT url, "Name", "날짜:Date:start"
FROM "{탐색된_collection_id}"
WHERE "날짜:Date:start" = '{대상_날짜}'
```

⚠️ 컬럼명은 인라인 DB마다 다를 수 있어요. `notion-fetch`로 스키마를 먼저 확인하고, 날짜 관련 컬럼을 동적으로 탐색해요.

이미 해당 날짜 기록이 있으면:
- 사용자에게 "이미 {날짜} 기록이 있어요. 덮어쓸까요?" 확인
- 확인 없이 덮어쓰지 않음

### 항목 생성

`notion-create-pages`로 작업 트래커 항목을 생성해요:

```
notion-create-pages:
  parent: { data_source_id: "{탐색된_collection_id}" }
  pages: [{
    properties: {
      "Name": "{대상_날짜} 작업 기록",
      // 날짜 property가 있으면 설정
    },
    content: "{마크다운 본문}"
  }]
```

### 본문 포맷

```markdown
## 완료
- 완료된 작업 항목1
- 완료된 작업 항목2

## 진행중
- 진행중인 작업 항목

## 참고
- Notes DB에서 가져온 보충 정보 (있는 경우만)
```

간결하게 작성해요. Daily DB의 체크박스 항목을 그대로 가져오되, 불필요한 세부사항은 생략해요.

## 6단계: 결과 반환

처리 결과를 요약해서 보여줘요:

```
작업 트래커 기록 완료!

✅ Create/Variation 모델 학습 및 검증
  - 완료 3건, 진행중 1건
  - 작업 트래커: {notion_page_url}

✅ QA 자동화 구축
  - 완료 2건
  - 작업 트래커: {notion_page_url}

⚠️ 매칭 실패: [프론트엔드 리팩토링] → 통합 태스크 DB에 해당 태스크 없음
```

## 주의사항

- 노션 MCP 연결이 안 되어 있으면 사용자에게 연결 필요를 안내해요
- 페이지 접근 권한 오류 시 Connection 추가 필요를 안내해요
- **회사 태스크 페이지의 기존 내용은 절대 수정하지 않아요** — 인라인 DB에 항목 추가만 해요
- 중복 기록 방지를 위해 반드시 기존 항목 확인 후 생성해요
- 사용자 확인 없이 민감한 정보(API 키, 비밀번호 등)를 작성하지 않아요
- 인라인 DB 스키마가 태스크마다 다를 수 있으므로, 매번 동적으로 확인해요
- Notion API 타임아웃 시 최대 2회 재시도해요

## 구현 예시: 작업 흐름

### 입력 예시
사용자: "2024-03-15 작업 트래커 작성해줘"

### 처리 흐름

1. **대상 날짜 결정**
   - 사용자 입력: `2024-03-15` ✓

2. **Daily 페이지 조회**
   ```
   SELECT url, "Name", "date:Date:start"
   FROM "collection://31119aac-27e4-8008-bb4b-000bd838f764"
   WHERE "date:Date:start" = '2024-03-15'
   ```
   - 결과: Daily/2024-03-15 페이지 발견

3. **Daily 페이지 파싱**
   ```markdown
   ## todo
   * [Create/Variation]
   	- [x] 모델 학습 데이터 준비
   	- [x] 학습 스크립트 작성
   	- [ ] 모델 성능 테스트
   * [QA 자동화]
   	- [x] 테스트 케이스 작성
   ### misc.
   - 회의 참석
   ```
   - 추출 결과:
     - `Create/Variation`: 완료 2건, 진행중 1건
     - `QA 자동화`: 완료 1건

4. **Notes DB 보조 정보**
   ```
   SELECT url, "Name", "생성일시:Date:start"
   FROM "collection://31119aac-27e4-80d9-a83c-000be57af066"
   WHERE "생성일시:Date:start" = '2024-03-15'
   ```
   - 결과: Notes 없음

5. **통합 태스크 매칭**
   ```
   SELECT url, "태스크명", "상태", "담당자"
   FROM "collection://db1ed8bf-4edf-4a87-b5bd-8774ce361a73"
   WHERE "상태" NOT IN ('완료', '아카이빙')
   ```
   - 결과:
     - `Create/Variation 모델 학습 및 검증` (담당자: 김하영) ✓
     - `QA 자동화 구축` (담당자: 김하영) ✓

6. **작업 트래커 DB 탐색**
   - Create/Variation 태스크 페이지 → 인라인 DB 발견: `collection://xyz...`
   - QA 자동화 태스크 페이지 → 인라인 DB 발견: `collection://abc...`

7. **중복 확인 및 생성**
   - Create/Variation 작업 트래커:
     ```
     SELECT url FROM "collection://xyz..."
     WHERE "날짜:Date:start" = '2024-03-15'
     ```
     - 기존 기록 없음 → 생성 진행

   - 생성 항목:
     ```
     Name: 2024-03-15 작업 기록
     Content:
     ## 완료
     - 모델 학습 데이터 준비
     - 학습 스크립트 작성

     ## 진행중
     - 모델 성능 테스트
     ```

   - QA 자동화 작업 트래커:
     ```
     SELECT url FROM "collection://abc..."
     WHERE "날짜:Date:start" = '2024-03-15'
     ```
     - 기존 기록 없음 → 생성 진행

   - 생성 항목:
     ```
     Name: 2024-03-15 작업 기록
     Content:
     ## 완료
     - 테스트 케이스 작성
     ```

8. **결과 반환**
   ```
   작업 트래커 기록 완료!

   ✅ Create/Variation 모델 학습 및 검증
     - 완료 2건, 진행중 1건
     - 작업 트래커: https://notion.so/...

   ✅ QA 자동화 구축
     - 완료 1건
     - 작업 트래커: https://notion.so/...
   ```

## 에러 처리

### 시나리오별 대응

#### Daily 페이지가 없는 경우
```
❌ 2024-03-15의 Daily 페이지를 찾을 수 없어요.
Daily DB(SSoT)를 확인해주세요.
```

#### 통합 태스크 매칭 실패
```
⚠️ 다음 Daily 항목은 통합 태스크 DB에 매칭되지 않았어요:
- [프론트엔드 리팩토링]
- [기술 부채 정리]

통합 태스크를 추가하거나, Daily 레이블을 수정해주세요.
```

#### 인라인 DB를 찾지 못한 경우
```
⚠️ Create/Variation 태스크 페이지에서 '데일리 작업 기록' DB를 찾을 수 없어요.
태스크 페이지에 인라인 DB를 추가해주세요.

[태스크 페이지로 이동] {url}
```

#### 노션 연결 오류
```
❌ Notion MCP 연결이 필요해요.
Claude Code의 설정에서 "Notion" 커넥터를 추가해주세요.
```

#### 이미 기록된 경우
```
ℹ️ 2024-03-15의 Create/Variation 작업 기록이 이미 있어요.
덮어쓸까요?

[예] [아니오]
```

## 추가 기능 (향후 확장)

- **자동 스케줄링**: 매일 자동으로 그날의 Daily를 처리
- **주간 요약**: 일주일 기록을 자동 집계
- **태스크별 통계**: 완료율, 시간 추적 등
- **Slack 알림**: 작업 트래커 생성 시 팀 채널에 공유
- **템플릿 커스터마이징**: 본문 포맷 사용자 설정

## 참고사항

- 이 스킬은 **읽기 전용** 권한으로 개인 DB에 접근해요
- **쓰기 권한**은 회사 태스크 페이지의 인라인 DB에만 사용해요
- Daily DB는 SSoT로 변경되지 않아요
- 각 실행은 독립적이므로 이전 실행 기록은 유지해요
