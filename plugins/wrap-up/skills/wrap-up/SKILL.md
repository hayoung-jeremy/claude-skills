# Wrap-Up: 작업 내용 노션 정리 스킬

## 역할

사용자의 작업 내용을 분석하고, 노션에 정리 문서를 생성해요.

## 0단계: 생성 위치 결정

아래 우선순위로 노션 페이지 생성 위치를 결정해요:

### 우선순위 1 — 사용자가 URL을 명시한 경우

사용자가 노션 URL을 직접 제공하면 무조건 해당 위치에 생성해요.
- `notion-fetch`로 URL의 타입을 확인해요
- 데이터베이스 → 해당 DB의 data_source_id로 페이지 생성
- 일반 페이지 → 해당 페이지 하위에 생성

### 우선순위 2 — 개인 설정 파일이 있는 경우

`~/.claude/wrap-up.json` 파일에 저장된 기본 위치를 사용해요.

파일 형식:
```json
{
  "default_data_source_id": "31119aac-27e4-80d9-a83c-000be57af066",
  "default_location_name": "Notes"
}
```

### 우선순위 3 — 설정 파일이 없는 경우 (첫 사용)

사용자에게 질문해요:

> "노션 정리 문서를 생성할 기본 위치를 설정해주세요.
> 노션 데이터베이스 또는 페이지 URL을 알려주시면, 앞으로 자동으로 거기에 생성할게요."

사용자가 URL을 제공하면:
1. `notion-fetch`로 해당 위치 정보를 확인해요
2. 데이터베이스면 data_source_id 추출, 페이지면 page_id 추출
3. `~/.claude/wrap-up.json`에 저장해요
4. 이후 작업을 계속 진행해요

## 1단계: 정리 대상 파악

아래 우선순위로 정리 대상을 결정해요:

1. **사용자가 명시적으로 지정한 경우** — 해당 대상만 정리
   - 예: "feat/2.5d-edit 브랜치 작업 정리해줘" → 해당 브랜치 분석
   - 예: "오늘 커밋 정리해줘" → 오늘 날짜 커밋만 분석
   - 예: "이번 대화 내용 정리해줘" → 대화 컨텍스트 기준
2. **$ARGUMENTS가 있는 경우** — 해당 내용 기준으로 분석
3. **아무 지정 없는 경우** — 현재 대화 컨텍스트에서 수행한 작업 기준

### 브랜치/커밋 분석이 필요한 경우

- 현재 브랜치 확인: `git branch --show-current`
- 리모트 브랜치라면: `git fetch origin <branch>` 먼저 실행
- 오늘 커밋 필터: `git log --since="today" --format="%h %ad %s" --date=short`
- 해당 브랜치 고유 커밋만 추출 (main/prod 대비 diff)
- 변경 파일 확인: `git diff <base>...<target> --stat`
- 핵심 파일 내용은 `git show <branch>:<filepath>`로 읽어요 (체크아웃 불필요)

## 2단계: 문서 구조 생성

노션 페이지 내용을 아래 형태의 마크다운으로 구성해요:

```
## 요약
(1~3줄로 작업 전체 요약)

## 작업 내용
- [x] 완료된 작업 항목
- [ ] 미완료 작업 항목
  - [x] 세부 항목

## 상세 노트
(필요시 기술적 설명, 파일 구조, 디버깅 이력 등)
```

### 작성 규칙
- 제목(Name)은 작업 내용을 요약한 간결한 한 줄로 작성해요
- 본문은 노션 마크다운 형식을 사용해요
- todo 항목은 체크박스(`- [x]`, `- [ ]`)를 사용해요
- 코드블록, 인라인 코드 등을 활용해요
- 불필요하게 길지 않게, 핵심 위주로 정리해요

## 3단계: 노션 페이지 생성

0단계에서 결정된 위치에 페이지를 생성해요.

### 데이터베이스에 생성하는 경우

```
notion-create-pages:
  parent: { data_source_id: "<결정된 data_source_id>" }
  pages: [{ properties: { "Name": "<정리 제목>" }, content: "<마크다운 본문>" }]
```

주의: 데이터베이스마다 프로퍼티 이름이 다를 수 있어요. `notion-fetch`로 스키마를 먼저 확인하고, title 타입 프로퍼티의 실제 이름을 사용해야 해요.

### 일반 페이지 하위에 생성하는 경우

```
notion-create-pages:
  parent: { page_id: "<결정된 page_id>" }
  pages: [{ properties: { "title": "<정리 제목>" }, content: "<마크다운 본문>" }]
```

### 아이콘 설정

페이지 생성 후, 아이콘을 "claude" 커스텀 이모지로 설정해야 해요.
현재 API 제약으로 자동 설정이 불가할 수 있으므로, 불가한 경우 사용자에게 안내해요.

## 4단계: 결과 반환

- 생성된 노션 페이지 URL을 사용자에게 표시해요
- 정리한 내용의 간략한 요약도 함께 표시해요

예시:
```
노션에 정리 완료했어요!
- 페이지: [2.5D 이미지 편집기 MVP 구현](https://notion.so/xxx)
- 위치: Notes DB
- 항목: 6개 기능 구현 완료, 4개 버그 수정
```

## 주의사항

- 노션 MCP 연결이 안 되어 있으면 사용자에게 연결 필요를 안내해요
- 페이지 접근 권한 오류 시 Connection 추가 필요를 안내해요
- 기존 페이지 덮어쓰기는 금지 — 항상 새 페이지를 생성해요
- 사용자 확인 없이 민감한 정보(API 키, 비밀번호 등)를 노션에 작성하지 않아요
